Belinda UI

If I were to create a UI for Belinda, this is what I would do.

For fun, let's make this a tutorial project. We'll start one step at a time with basics and grow the project from there, organically.

Define Data Requirements

Popups take three parameters. Nine altogether, because the third is multi-part (has up to 7 members).

The basic call is:


window.open( url, target, options ) := Window

All three are optional.

Blank Slate (no URL)

If no arguments are specified, the typical browser response is to create an empty window (usually this resides in a new browser tab). This tab as a location (URL) of about:blank.

It's up to the app to then manually write content into this new window's document. We'll discuss that option later.

If a URL is specified, the browsers tries to open it for display. This may result in any of the errors we see (e.g. 403, 404, and so on).

Targets

Targets are window names. These are used to indicate to the browser which window (tab, popup, etc) a page should be loaded into.

Oftentimes, especially when a single popup or tab is needed and will be reused, this parameter is set to "_blank". There's no rule about this that I'm aware of, just common practice.

I don't know for certain, but presumably the browser consider an empty/invalid/null/undefined parameter here as all referring to a single default object.

Options

Now we get to the real purpose of this exercise. Coming up with a URL and unique target name are simple. This one is a bit more complicated.

The options parameter consists of four parts. Again, this can be expanded to seven parts since one is compound.

  • Rectangle Location
  • Show Scrollbars? (yes/no)
  • Menu? (yes/no)
  • Resizable (yes/no)

Let's break these down in a table, for future reference.

Name Type Meaning
left Number X-coord for top-left corner
top " Y-coord for top-left corner
width " Window width
height " Window height
resizable Yes/No Allow resizing?
scrollbars Yes/No Show scrollbars
toolbars Yes/No Show toolbars

Defaults

If no options are specified, the normal default is for the browser to fall back on using a new tab, rather than a popup. You need to at least specify top and left (the corner point; also call the anchor point).

The version of Chrome that I'm using assumes yes for any boolean (yes/no) parameter not provided. I have no idea if this applies universally or varies by browser.

So, a minimal example is in order.

Example

Let's say we just want to popup a blank window. Nothing fancy. No content. Just a blank form. We'll try it now.

This ony works in the browser. Not ReText.


options = "left=100; top=100";
window.open( "", "_blank", options );




// Han Solo ~ Safe Callbacks

function han( falcon, cargo, options ) {
    try {
        return falcon( cargo );
    } catch( empire ) {
        if ( options ) {
          if ( options.alert   ) { alert( empire );         }
          if ( options.console ) { console.error( empire ); }
        }
        return empire;
    }
}
